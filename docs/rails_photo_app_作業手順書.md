# Rails課題 作業手順書

## 目的
この手順書は、写真管理アプリケーション（OAuth連携付き）を、要件を満たしつつ過不足なく実装するための作業順序を整理したもの。

---

## 0. ゴールの確認
最終的に、以下がローカル環境で動作している状態を目指す。

- メールアドレス / パスワードでログインできる
- ログインユーザーが自分の写真一覧を見られる
- 写真をアップロードできる
- ログアウトできる
- OAuth認可画面に遷移できる
- 認可コードからアクセストークンを取得し、セッションに保存できる
- 写真タイトルと画像URLを連携アプリへ投稿できる
- READMEに起動方法と補足を記載できる

---

- OAuth連携テストアプリのログイン用ユーザーID
- パスワード

### 1-3. OAuth連携テストアプリの疎通確認
先に外部アプリへアクセスし、以下を確認する。

- ログインページが表示される

- ログイン後画面が表示される

## 2. 開発環境の準備

### 2-1. Railsアプリ作成
- `rails new` で新規アプリを作成
- DB は扱いやすいものを選ぶ（SQLite で十分）
- Git リポジトリを初期化し、最初のコミットを作る

### 2-2. 実装方針の整理
最初に以下の構成を想定しておく。

- User: ログイン用ユーザー
- Photo: 写真データ
- SessionsController: ログイン / ログアウト
- PhotosController: 一覧 / 新規作成
- OauthController: 認可コールバック処理
- TweetsController: 外部投稿処理

## 3. データ設計

### 3-1. User モデル
最低限必要な項目を決める。

- email
- password_digest

想定:
- `has_secure_password` を使う
- メールアドレスは保存時に正規化しておくと扱いやすい

### 3-2. Photo モデル
最低限必要な項目を決める。

- title
- user_id
- image

想定:
- 1人のユーザーが複数の写真を持つ
- 画像アップロードは Rails 標準機能で扱えるならそれを優先する

### 3-3. 関連
- User has_many :photos
- Photo belongs_to :user

## 4. 認証の土台を作る

### 4-1. ユーザー作成方法を決める
課題上、ユーザー登録画面は不要なので以下のどちらかで用意する。

- seeds.rb
- console / DB 直投入

### 4-2. ログイン処理を実装
要件:
- メールアドレス入力
- パスワード入力
- 大文字小文字を無視してメールアドレス照合
- 成功時は写真一覧へ遷移
- 失敗時はエラー付きでログイン画面再表示
- 再表示時はメールアドレスを保持

### 4-3. セッション管理を実装
- ログイン成功時に `session[:user_id]` を保存
- 未ログイン時は保護ページへ入れないようにする
- `current_user` と `logged_in?` のような補助メソッドを ApplicationController か helper に用意する

### 5-1. 一覧ページを実装

### 5-3. 未ログインアクセス制御

## 8. OAuth認可画面への遷移を実装

### 8-0. 課題仕様で求められている内容
写真一覧ページに、外部OAuth連携テストアプリの認可画面を開くリンクを表示する。クリック時は OAuth 2.0 Authorization Code Flow の認可リクエストとして外部サービスへ遷移させる。

### 使用する認可エンドポイント
- http://unifa-recruit-my-tweet-app.ap-northeast-1.elasticbeanstalk.com/oauth/authorize

### 必須クエリパラメータ
- client_id: 支給された client_id
- response_type: code
- redirect_uri: http://localhost:3000/oauth/callback
- scope: write_tweet
- state: 指定不要（課題文ではなし）


### 8-1. 一覧ページに認可リンクを配置
リンク押下で外部アプリの認可画面へ遷移させる。

### 8-2. 認可URLを組み立てる
必要パラメータ:
- client_id
- response_type=code
- redirect_uri=http://localhost:3000/oauth/callback
- scope=write_tweet

### 8-3. 動作確認
- クリックで外部認可画面が開く
- 承認ボタンが見える

---

## 9. OAuthコールバックを実装

### 9-0. 課題仕様で求められている内容
認可画面で承認後、以下URLへリダイレクトされる。
- GET http://localhost:3000/oauth/callback?code=xxxxxxxx

この code を使ってアクセストークンを取得し、セッションへ保存する。否認時の対応は不要。

### トークン取得エンドポイント
- POST http://unifa-recruit-my-tweet-app.ap-northeast-1.elasticbeanstalk.com/oauth/token

### 送信パラメータ
- code: コールバックで受け取った認可コード
- client_id: 支給された client_id
- client_secret: 支給された client_secret
- redirect_uri: http://localhost:3000/oauth/callback
- grant_type: authorization_code

### レスポンス
JSON形式。access_token を取り出して利用する。
アクセストークンに有効期限はないため、一度取得すれば継続利用可能。

### 9-1. コールバック用ルーティング作成
- `GET /oauth/callback`

### 9-2. 認可コードを受け取る
- クエリパラメータ `code` を受け取る
- エラーにならずに処理できるようにする

### 9-3. アクセストークン取得処理
外部アプリの token エンドポイントに POST し、以下を送る。

- code
- client_id
- client_secret
- redirect_uri
- grant_type=authorization_code

### 9-4. トークンを保存
- レスポンスJSONから access_token を取得
- `session[:access_token]` に保存
- 写真一覧へリダイレクト

---

## 10. ツイート投稿機能を実装

### 10-0. 課題仕様で求められている内容
アクセストークン取得後、各写真に「ツイートする」ボタンを表示し、外部APIへ投稿する。

### 投稿先エンドポイント
- POST http://unifa-recruit-my-tweet-app.ap-northeast-1.elasticbeanstalk.com/api/tweets

### HTTPヘッダー
- Content-Type: application/json
- Authorization: Bearer {access_token}

### リクエストBody
JSON形式:
- text: 写真タイトル
- url: 画像URL（imgタグで表示しているURL。localhost:3000でOK）

### 成功条件
- HTTPステータス 201 を成功扱いとする
- 成功後は写真一覧へリダイレクトする


### 10-1. ボタン表示条件
- `session[:access_token]` があるときだけ、各写真に「ツイートする」ボタンを表示する

### 10-2. 投稿処理を実装
外部APIへ POST する内容:
- text: 写真タイトル
- url: 画像URL

HTTP条件:
- Content-Type: application/json
- Authorization: Bearer アクセストークン

### 10-3. 成功時の挙動
- ステータスコード 201 を成功扱いにする
- 写真一覧へ戻す

### 10-4. URLの扱い
- 画像URLは `img` タグに指定されているURLでOK
- localhost のURLで問題ない
- 必要なら URL helper を使って絶対URLを生成する
